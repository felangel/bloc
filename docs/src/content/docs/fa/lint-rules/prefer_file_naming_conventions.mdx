ترجیح داده می‌شود از قراردادهای نام‌گذاری فایل پیروی کنید.

:::note

این قانون lint در نسخهٔ `0.3.0` از
[`package:bloc_lint`](https://pub.dev/packages/bloc_lint) معرفی شده است

:::

## دلیل

برای حفظ یکپارچگی، سهولت نگهداری و جداسازی مسئولیت‌ها، ترجیح دهید نمونه‌های
`bloc` و `cubit` را در فایل‌های Dart مربوط به خودشان تعریف کنید به‌جای آنکه
آن‌ها را به‌صورت درون‌خطی قرار دهید.

:::tip

در نظر داشته باشید از دستور `bloc new <component>` از
[package:bloc_tools](https://pub.dev/packages/bloc_tools) برای تولید سریع و
یکنواخت نمونه‌های جدید bloc/cubit استفاده کنید.

:::
