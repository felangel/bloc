За замовчуванням лінтер bloc не видаватиме жодної діагностики, якщо ви явно не
налаштували параметри аналізу проєкту.

Для початку створіть або змініть наявний файл `analysis_options.yaml` у корені
вашого проєкту, щоб включити список правил під ключем верхнього рівня bloc:

<BlocLintBasicAnalysisOptionsSnippet />

Запустіть лінтер за допомогою наступної команди в терміналі:

<RunBlocLintInCurrentDirectorySnippet />

Наведена вище команда проаналізує всі файли в поточному каталозі та його
підкаталогах, але ви також можете перевірити конкретні файли та каталоги,
передавши їх як аргументи командного рядка:

<RunBlocLintInSrcTestSnippet />

Наведена вище команда проаналізує весь код у каталогах `src` та `test`.

Якщо правило `avoid_flutter_imports` увімкнено, будь-який файл bloc або cubit,
що містить імпорт flutter, буде позначено як попередження:

<AvoidFlutterImportsWarningSnippet />

Ви можете побачити попередження, запустивши команду `bloc lint`:

<RunBlocLintCounterCubitSnippet />

Вивід має виглядати так:

<AvoidFlutterImportsWarningOutputSnippet />
