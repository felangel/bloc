Надавайте перевагу використанню розширень `BuildContext` для доступу до
екземпляра `Bloc` або `Repository`.

:::note

Це правило лінтера було введено у версії `0.3.2` пакета
[`package:bloc_lint`](https://pub.dev/packages/bloc_lint)

:::

## Обґрунтування

Для узгодженості надавайте перевагу використанню розширень `BuildContext`, таких
як `context.read`, `context.watch` та `context.select`, замість
`BlocProvider.of`, `RepositoryProvider.of`, `BlocBuilder` або `BlocSelector`.

| явний метод                                                          | розширення            |
| -------------------------------------------------------------------- | --------------------- |
| `BlocProvider.of<Bloc>(context, listen: false)`                      | `context.read<Bloc>`  |
| `BlocBuilder<Bloc, State>(...)` або `BlocProvider.of<Bloc>(context)` | `context.watch<Bloc>` |
| `BlocSelector<Bloc, State>(...)`                                     | `context.select`      |

## Приклади

**Уникайте** використання `BlocProvider.of<T>(context)` для доступу до
екземпляра `Bloc`.
