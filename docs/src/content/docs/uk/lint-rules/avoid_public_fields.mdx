Уникайте надання публічних полів в екземплярах `Bloc` та `Cubit`.

## Обґрунтування

Компоненти бізнес-логіки підтримують власний `state` та видають зміни стану
через API `emit`. У результаті весь публічно доступний стан повинен бути наданий
через об'єкт `state`.

## Приклади

**НЕ НАДАВАЙТЕ** публічні поля в екземплярах bloc та cubit.

`analysis_options.yaml` у розділі `bloc` > `rules`:
