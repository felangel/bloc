Уникайте використання розширень `BuildContext` для доступу до екземплярів
`Bloc` або `Cubit`.

:::note

Це правило лінтера було введено у версії `0.3.0` пакета
[`package:bloc_lint`](https://pub.dev/packages/bloc_lint)

:::

## Обґрунтування

Для узгодженості та заради явності краще використовувати безпосередньо базові
методи замість розширень `BuildContext`. Це також корисно для тестування,
оскільки неможливо замокати метод розширення.

| розширення       | явний метод                                                          |
| ---------------- | -------------------------------------------------------------------- |
| `context.read`   | `BlocProvider.of<Bloc>(context, listen: false)`                      |
| `context.watch`  | `BlocBuilder<Bloc, State>(...)` або `BlocProvider.of<Bloc>(context)` |
| `context.select` | `BlocSelector<Bloc, State>(...)`                                     |

## Приклади

**Уникайте** використання розширень `BuildContext` для взаємодії з
екземплярами `Bloc` або `Cubit`.
