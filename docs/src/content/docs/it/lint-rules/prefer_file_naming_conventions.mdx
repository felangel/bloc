Preferisci seguire le convenzioni di nomenclatura dei file.

:::note

Questa regola di lint è stata introdotta nella versione `0.3.0` di
[`package:bloc_lint`](https://pub.dev/packages/bloc_lint)

:::

## Motivazione

Per coerenza, facilità di manutenzione e separazione delle responsabilità
preferisci definire istanze bloc e cubit nei rispettivi file Dart invece di
inserirle inline.

:::tip

Considera di usare il comando `bloc new <component>` da
[package:bloc_tools](https://pub.dev/packages/bloc_tools) per generare
rapidamente e coerentemente nuove istanze bloc/cubit.

:::

## Esempi

**Preferisci** dichiarare istanze bloc/cubit nei loro rispettivi file.
