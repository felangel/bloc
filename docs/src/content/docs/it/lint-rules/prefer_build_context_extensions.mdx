Preferisci usare estensioni `BuildContext` per accedere a un'istanza di `Bloc` o `Repository`.

:::note

Questa regola di lint Ã¨ stata introdotta nella versione `0.3.2` di
[`package:bloc_lint`](https://pub.dev/packages/bloc_lint)

:::

## Motivazione

Per coerenza, preferisci usare estensioni `BuildContext` come `context.read`,
`context.watch`, e `context.select` invece di `BlocProvider.of`,
`RepositoryProvider.of`, `BlocBuilder` o `BlocSelector`.

| metodo esplicito                                                     | estensione             |
| ------------------------------------------------------------------- | --------------------- |
| `BlocProvider.of<Bloc>(context, listen: false)`                     | `context.read<Bloc>`  |
| `BlocBuilder<Bloc, State>(...)` o `BlocProvider.of<Bloc>(context)` | `context.watch<Bloc>` |
| `BlocSelector<Bloc, State>(...)`                                    | `context.select`      |

## Esempi

**Evita** di usare `BlocProvider.of<T>(context)` per accedere a un'istanza di `Bloc`.
