Avoid using context extensions to access `Bloc` or `Cubit` instances.

## Rationale

Prefer using `BlocProvider.of<T>(context)` over context extensions like
`context.read<BlocType>()`, `context.watch<BlocType>()`, and `context.select<BlocType, R>(...)` for
explicit dependency injection and better testability. This rule only applies to types ending with 'Bloc' or 'Cubit'.
